# Birmingham Water Works Board — Home Assistant Integration

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-orange.svg)](https://github.com/hacs/integration)

Track your BWWB water usage directly in Home Assistant. Exposes cumulative meter reads, last read date, and billing-period consumption as sensors — compatible with the HA Energy dashboard (water consumption).

---

## Why This Integration Exists

There is no official API for BWWB. This integration reverse-engineers the customer portal at [web.bwwb.org](https://web.bwwb.org) to extract meter reading data from the SAP IS-U / OData backend.

## Why a Sidecar Auth Service Is Required

BWWB uses **Utegration Mobius** — a SAP IS-U frontend built on SAPUI5. The login page renders **entirely in JavaScript**: field rendering, hidden XSRF token injection, and the SAP SSO handshake (which sets the `MYSAPSSO2` session cookie) all happen client-side via the SAPUI5 framework.

Attempting to log in with a standard HTTP client (aiohttp, requests, curl) returns **403 Forbidden** because:
1. The SAP ITS endpoint requires an XSRF token generated by the JS-rendered form
2. The SAP SSO cookie (`MYSAPSSO2`) is only issued after the full SAPUI5 flow completes
3. Direct HTTP POST bypasses the JS handshake entirely

**The solution:** A small Playwright-based authentication service runs alongside Home Assistant (on the same Pi/server). The HA integration calls this service to perform the JavaScript login and fetch OData data. Sessions are cached for 10 minutes on the sidecar side.

## Architecture

```
┌─────────────────────────────┐
│   Home Assistant (Docker)   │
│                             │
│  bwwb_water integration     │
│    └── fetch() ─────────────┼──► POST http://<auth-service-ip>:18792/bwwb/data
│                             │         (Playwright on Pi, ~5-10s first auth)
│                             │         (cached 10min, fast on subsequent calls)
└─────────────────────────────┘

┌─────────────────────────────┐
│   Pi / Local Server         │
│                             │
│  utility-auth-service.py    │
│    ├── /bwwb/data  ◄────────┼── HA data requests (login + OData fetch)
│    ├── /sc/auth    ◄────────┼── Southern Company (shared service)
│    └── /health              │
│                             │
│  Playwright + Chromium      │
│    └── Fills SAPUI5 form    │
│        Harvests cookies     │
│        Fetches OData        │
└─────────────────────────────┘
```

## Prerequisites

### 1. Auth Sidecar Service

The auth service must run on a machine that can reach both:
- Home Assistant (to receive requests)
- `web.bwwb.org` (to perform the login)

**Install dependencies:**
```bash
pip3 install playwright aiohttp
# Run on the machine where the sidecar will be hosted:
python3 -m playwright install chromium
```

**Download and run the service:**
```bash
# Download
curl -o utility-auth-service.py https://raw.githubusercontent.com/Space-C0wboy/ha-bwwb-water/main/auth-service/utility-auth-service.py

# Run (default bind: 0.0.0.0:18792)
python3 utility-auth-service.py
```

**Or run as a systemd service:**
```ini
[Unit]
Description=Utility Auth Service (Playwright)
After=network.target

[Service]
Type=simple
User=your-user
ExecStart=/usr/bin/python3 /path/to/utility-auth-service.py
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### 2. Network Access

The auth service listens on port `18792` by default. Home Assistant must be able to reach it. If HA is in Docker and the service is on a separate host, use that host's LAN IP.

If you use managed networking with firewall policies between VLANs, add a rule allowing HA → auth service on TCP port 18792.

## Installation

### HACS (Recommended)

1. In HACS → Integrations → ⋮ → Custom Repositories
2. Add `https://github.com/Space-C0wboy/ha-bwwb-water` as type **Integration**
3. Install "Birmingham Water Works Board"
4. Restart Home Assistant

### Manual

```bash
cp -r custom_components/bwwb_water /config/custom_components/
```
Restart Home Assistant.

---

## Configuration

1. Settings → Integrations → Add Integration → **Birmingham Water Works Board**
2. Enter your BWWB portal username and password
3. Enter the URL of your auth sidecar service (e.g. `http://192.168.1.100:18792`)

**Note:** First setup takes ~10–15 seconds while Playwright launches and logs in.

## Sensors

| Sensor | Entity ID | Unit | Device Class | State Class | Description |
|--------|-----------|------|--------------|-------------|-------------|
| BWWB Water Meter | `sensor.bwwb_water_meter` | ft³ | `water` | `total_increasing` | Cumulative meter read in cubic feet (CCF × 100). Primary sensor for HA Energy dashboard. |
| BWWB Water Meter (CCF) | `sensor.bwwb_water_meter_ccf` | CCF | — | `total_increasing` | Same cumulative read in CCF for convenience |
| BWWB Last Read Date | `sensor.bwwb_last_read_date` | — | — | — | Date of the most recent billing meter read |
| BWWB Current Period Usage | `sensor.bwwb_current_period_usage` | CCF | `water` | `total` | Water consumption for the current billing period |
| BWWB Previous Period Usage | `sensor.bwwb_previous_period_usage` | CCF | `water` | `total` | Water consumption for the previous billing period |
| BWWB Current Balance | `sensor.bwwb_current_balance` | USD | `monetary` | `total` | Current account balance due |
| BWWB Last Bill Amount | `sensor.bwwb_last_bill_amount` | USD | `monetary` | `measurement` | Amount of the most recent bill |
| BWWB Last Bill Due Date | `sensor.bwwb_last_bill_due_date` | — | — | — | Due date of the most recent bill |

> **Note:** Entity IDs may vary if you have renamed entities. The names above are the defaults.

### Energy Dashboard

Go to **Settings → Dashboards → Energy → Water consumption** and add `sensor.bwwb_water_meter`.

**Note:** BWWB meter reads are updated approximately monthly (billing cycle). Daily granularity is not available unless BWWB deploys AMI smart meters to your address.

## Data Notes

- **Granularity:** BWWB currently provides billing-period meter reads (~monthly). No daily or hourly data is available unless you have an AMI smart meter.
- **Units:** BWWB reports in CCF (hundreds of cubic feet). This integration converts to ft³ (multiply by 100) for HA Energy compatibility. Period usage sensors report in CCF directly.
- **SAP platform:** The OData service (`ZUTE_ERP_UT_UMC_SRV`) exposes `ContractConsumptionValues` for billing-period consumption history (up to 12 periods).

## Troubleshooting

**`cannot_connect` on setup**
- Verify the auth service is running: `curl http://<auth-service-ip>:18792/health`
- Check firewall rules if HA and the auth service are on different VLANs
- Check the auth service logs for Playwright errors

**`invalid_auth` on setup**
- Verify credentials at [web.bwwb.org](https://web.bwwb.org/myaccount/?sap-client=300&sap-language=EN)
- BWWB usernames are typically ALL CAPS

**No data / sensor unknown after setup**
- BWWB only updates meter reads on billing cycles. If your last read was weeks ago, the sensor will show that value until the next read.
- Check HA logs for OData errors
